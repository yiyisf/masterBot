FIGURE 02 · NEW · Identity Model

Identity & RBAC 三方授权模型

企业 agent 平台的核心安全模型：每一次 agent 操作都涉及人类用户、 Agent 身份、工具权限三方。 最终授权遵循 权限交集原则——既防止低权限用户用高权限 agent 越权，也防止高权限用户用 agent 在错误场景下被滥用。

① 人类用户身份

user_id
来自企业 IdP (SSO)
department
SCIM 同步 · HR/财务/工程...
roles[]
岗位角色 · 经理/员工/外包
groups[]
IdP groups · 数据访问域

② Agent 身份（非人类）

agent_id
每 agent 唯一加密 ID
agent_role
hr-bot/it-helper/finance-bot
tenant
归属租户 · 强隔离
capabilities[]
注册时声明的工具集
owner
Agent 维护人 · 必填

③ 工具/资源权限

tool_id
Tool Registry 中的版本化条目
required_scopes[]
e.g. crm:read, hr:write
data_classification
public/internal/confidential/restricted
approval_level
none/peer/manager/security
audit_severity
trace/info/warn/critical
allow = user.scopes ∩ agent.capabilities ∩ tool.required_scopes
权限交集原则 · 三方权限的逻辑交集必须非空 · 任一方缺失即拒绝 · 同时记录 user_id + agent_id + tool_id 至审计日志（"为什么允许 / 谁负责"）

实施关键：(1) 不要用 SSO + groups 替代 RBAC——前者是"应用登录"，后者是"工具执行"，必须分开。 (2) 每个 agent 都必须有一个明确的 human owner——出问题时责任归属。 (3) 三方权限的最终决策由 L4.5 Policy Engine 用策略即代码（OPA Rego）实现，避免散落在 agent 业务代码中。

FIGURE 03 · NEW · Skill Factory

Skill Factory 生产流水线

masterBot 的核心差异化能力——把 v1 现有的 Auto-Skill Generator 升级为 5 阶段生产流水线： Understand → Synthesize → Verify → Eval → Publish。 关键升级：每个生成的技能必须通过静态校验 + 安全扫描 + 沙箱测试 + 自动评估四道关，才能进入企业技能市场。

1
UNDERSTAND
挖掘真实需求
  • 读取用户描述 + 历史对话
  • 查询 Skill Catalog 防重复
  • 多轮反问澄清
  • 生成内部 Spec（YAML）
2
SYNTHESIZE
生成代码与文档
  • SKILL.md（描述+触发词）
  • scripts/*.ts 实现
  • references/ 资源
  • tests/ 单元测试
3
VERIFY
静态 + 安全双重校验
  • 结构验证（命名/元数据）
  • SAST 扫描（Semgrep）
  • 硬编码 key 检测
  • 依赖审计（SCA）
  • 权限范围分析
4
EVAL
沙箱评估
  • 隔离 Docker 沙箱
  • 自动生成 5+ 测试用例
  • LLM-as-Judge 评分
  • 性能 / 成本基线
  • 人工评审门
5
PUBLISH
发布上架
  • 注册到 Tool Registry
  • 触发 RBAC 默认策略
  • 添加到 Skill Catalog
  • 通知 owner & 申请人
  • 灰度 5% → 100%
▲ 自我进化闭环（Auto-Curator）
生产环境运行 30 天后，Auto-Curator 自动分析每个技能的使用率与成功率： 高频高成功率→ 推荐为「企业精选」； 低频但有价值→ 保留并优化描述（让更多 agent 发现）； 低频低成功率→ 自动归档（防止技能库膨胀）。 同时把负向反馈（用户点踩、错误重试模式）回流到 Stage 1 触发新一轮 Skill 生成。

对比 v1 现状：v1 的 Auto-Skill Generator 只有 Stage 1+2（描述→生成→热加载），缺少 3+4+5。 在企业场景下这是不可接受的——一个技能可能被几百员工调用，质量与安全必须有强保证。 Stage 3-5 的引入让 masterBot 从"个人玩具"晋升为"企业平台"。

FIGURE 04 · NEW · Skill Lifecycle

技能生命周期治理

技能不是「一次生成永久使用」，而是有完整生命周期的资产。 下图展示一个技能从「员工提需求」到「自动归档」的 8 个状态，以及每个状态之间的转换条件、触发者、审计要点。

📝 Drafting · 草稿期

员工通过对话或表单提出技能需求。Skill Factory Stage 1 (UNDERSTAND) 处于活跃状态，反复确认需求。

触发者：员工 · 持续：5min~1h · 审计：需求记录

🔨 Synthesizing · 生成中

Stage 2 LLM 生成 SKILL.md + 代码 + 测试。生成期间技能不可用，仅创建者可见。

触发者：系统 · 持续：30s~5min · 审计：生成 prompt + 输出

🔍 Pending Review · 待评审

Stage 3+4 完成（静态 + 沙箱评估通过）后进入此状态。需要 指定 reviewer 审批。

触发者：系统自动 · 持续：0~3 工作日 · 审计：评估报告

✅ Approved · 已审批

Reviewer 批准后进入。此时已注册到 Tool Registry，但仅创建者所在部门可用，5% 灰度。

触发者：Reviewer · 持续：1~7 天 · 审计：审批人 + 评审意见

🚀 Active · 生产可用

灰度指标稳定后晋升。全员或指定部门可见，Skill Catalog 正式收录。开始累计使用统计。

触发者：Owner / 系统 · 持续：长期 · 审计：所有调用

⚠️ Deprecated · 已废弃

有更好的替代品 / 底层 API 变更 / 安全漏洞。技能仍可用但在 Catalog 中标灰，agent 优先选其他。

触发者：Owner / Auto-Curator · 持续：30~90 天 · 审计：废弃原因

📦 Archived · 归档

30 天无调用 OR 已废弃满 90 天。从 Tool Registry 移除，只保留代码与执行历史以供审计回溯。

触发者：Auto-Curator · 持续：永久保留 · 审计：不可删除

🚨 Quarantined · 隔离

异常状态：发现安全问题、错误率突增、被举报滥用。立即停用，等待安全团队复核。

触发者：SecOps · 持续：立即 · 审计：事件根因 + 处置

关键设计原则： （1）所有状态转换必须可审计——审计日志可回答"这个技能何时由谁创建/审批/废弃，过去 30 天被谁调用了多少次"。 （2）Quarantined 状态有最高优先级——可由任何 SecOps 成员直接触发，无需走完整审批流程。 （3）Archived 状态永不删除——满足 SOC2/GDPR 审计追溯要求。

FIGURE 07 · NEW · Evaluation Pyramid

三层评估金字塔

企业 agent 平台的"Evals are not optional"原则。 不同于 v1 仅有的 90 个单元测试（regression），v2 必须建立三层评估体系： 大量离线测试（保护）、 中量影子流量（验证）、 少量灰度发布（金标准）。

1
Offline Regression Eval · 离线回归
每次代码变更（PR）自动运行 · vitest + promptfoo + 自定义 capability suite · 覆盖所有已发布 Skill的核心场景 · 目标：通过率 ≥99%
频率: PR触发
规模: 1000+ cases
决策: block merge
2
Online Shadow Traffic · 影子流量
将 5-10% 真实用户请求同时发给新旧版本 · 对比 token 消耗、工具选择正确性、最终回答相似度 · 不返回新版本结果给用户，仅用于评估
频率: 持续
规模: 1000~10K/day
决策: 阻止上线
3
Production Canary · 生产灰度
5% → 25% → 50% → 100% 的渐进式放量 · 每个阶段观察 24~72h 的用户满意度（thumbs up/down）+ 错误率 + 成本 · 异常自动回滚
频率: 每次发布
规模: 真实用户
决策: 自动回滚

关键洞察：v1 的 90 个测试都属于 Tier 1（regression），完全缺失 Tier 2 + 3。 在企业场景下这是致命短板——你无法在影响所有员工之前发现 prompt 改动的副作用。 v2 的 Skill Factory Stage 4 (EVAL) 自动产出 Tier 1 测试用例； Phase 1 引入的 Langfuse Datasets 支持 Tier 2 影子流量；FeatureFlag 系统支持 Tier 3 自动灰度。
