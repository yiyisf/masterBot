# To Spec

grilling 拷问结束后，把当前对需求的理解沉淀为结构化规格，供人工核验。

## 规格结构

- **目标**：这个需求解决什么问题，达成什么效果，一两句话讲清楚。
- **范围**：具体要做什么、明确不做什么（避免范围蔓延）。
- **验收**：怎样算做完，有哪些可验证的标准。

## 完成信号

规格写好后，通过标记块协议交给平台核验，不要在没有产出规格的情况下直接开始写代码：
