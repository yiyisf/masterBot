# To Tickets

to-spec 产出规格后，把它拆解成一组卡片（实现步骤），每张卡片：

- 范围足够小，能在一次 agent 会话里完成。
- 有明确的完成标准，做完能独立验证。
- 卡片之间**只有先后顺序**，不建依赖图——输出顺序就是执行顺序（串行、共用同一个 worktree）。

## 完成信号

拆好后按执行顺序列出卡片标题，通过标记块协议交给平台：

```cmaster:done
{"cards":[{"title":"..."},{"title":"..."}]}
```

不要在这一步开始写代码，卡片列表核验通过后才进入实现阶段。
