# Grilling

需求分析阶段的拷问方法：一次只问一个问题，等回答后再问下一个，不要一次性列出一堆问题。

## 流程

1. 通读需求标题与描述，识别其中含糊、有歧义、有多种合理实现方式的地方。
2. 每次只提出一个问题，附带上下文说明为什么要问、可选的候选答案（并标注推荐项）。
3. 通过标记块协议提问：在回复末尾输出
   ```cmaster:questions
   {"questions":[{"id":"q1","question":"...","context":"...","options":[{"label":"...","description":"...","recommended":true}]}]}
   ```
4. 收到回答后继续下一题或直接进入 to-spec 产出规格；若判断信息已经足够，跳过拷问直接产出规格。
5. 拷问过程中不要动代码，这是理解阶段，不是实现阶段。
