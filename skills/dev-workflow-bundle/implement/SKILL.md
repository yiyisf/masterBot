# Implement

实现当前这一张卡片描述的改动，范围严格限定在卡片本身，不要顺手实现其他卡片的内容。

## 流程

1. 读卡片标题/描述，必要时先看一眼相关现有代码，理解改动落点。
2. 实现改动，配套测试（沿用仓库现有测试模式，不要另起一套）。
3. 跑测试、类型检查确认改动不破坏现状。
4. 提交代码（commit），不要单独为这张卡片开 PR——多张卡片共用同一个 worktree，全部完成后由平台统一开一个 PR。
5. 若卡片描述有歧义、或需要做只有人类才能决定的选择，通过标记块协议提问：

   ```cmaster:questions
   {"questions":[{"id":"q1","question":"..."}]}
   ```

6. 卡片实现完成后输出：

   ```cmaster:done
   {}
   ```
